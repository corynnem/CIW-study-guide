Intro to web development

  1. Internet vs Intranet
    • Internet: The Internet is a global network that uses TCP / IP protocols to connect millions of computers worldwide. It can include different networks, such as private, public, organizational, academic, government, etc. It allows users to communicate with each other and share massive amounts of data in various formats.
    • Intranet:The internet is a network created by connecting a large number of computers or computer networks. It is a type of network that cannot be owned by an entity. It is formed using networks of networks configured using complex standards and rules (such as protocols).

  2. Mobile websites vs mobile apps

    • Mobile Websites

      • Easy access
      • Developed the same regardless of device
      • Less performant than native apps
      • Find on search engine
      • Development and hosting costs
      • Weak offline capabilities

    • Mobile Apps

      • Apps come from the appstore
      • Each platform has a different language
      • More performant
      • Access to physical phone modules (RFID, GPS, etc)
      • Development, app store and server hosting costs
  3. Web dev trifecta
    • HTML5: structure and markup
    • CSS3: style and formatting
    • Javascript: dynamic interaction
  4. History of HTML

    • HTML3

      • Introduction of CSS (big stepping stone!)

    • HTML5

      • The video element
        • Attributes
          • source tag: the video source to be played
            • src: location of source
            • type: video encoding (video/mp4)
          • autoplay: a boolean
          • autopictureinpicture: If true allows for the hulu thing where you can watch outside of window
          • controls: if present allows the browser to provide controls to the video (volume, pause/resume playback)
          • controlsList: specify controlls, allowed are nodounload nofullscreen noremoteplayback
          • cross origin: whether or not to use CORS to fetch video
            • anonymous: sends cross origin request without a credential; usage restricted
            • useCredentials: sends cross origin request with a credential; usage restricted
          • height and width
          • loop: boolean
          • poster: a url for an image to be viewed while video is downloading
      • The audio element
        • Attributes
          • All the same as video
        • Drag and Drop API
          • First you need to make the element draggable by setting the draggable attribute to be true
          • The dataTransfer.setData() method gets the type and value of the data set to be draggable
          • The ondragover event specifies where dragged data can be dropped and accepts a function
        • Image captioning, download progress indicator
        • Offline browser storage
        • Geolocation access
          • navigator.geolocation()
  5. Semantic elements

    • Used for screen readers to let those who need them, know important sections of a page

    • Structural Semantic elements

      • header tag: Top of the document
      • main tag: The main content of the document
      • nav tag: The navigation menu
      • section tag: Portions of the document split up, like a div but more specific
      • article tag: Content parallel to the main content i.e. News article, blog, image
      • aside tag: Contend aside from the main content i.e. ad's/newsfeed
      • footer tag: Bottom of the webpage

    • Other Semantic elements

      • details tag: Additional details the user can view/hide
      • figcaption: caption for a figure tag
      • figure tag: diagrams, photos, code snippets
      • The mark tag: highlighted or marked text
  6. How to's in web development

    • If your code follows W3 standards carefully it will

      • Always run correctly in new or old browsers
      • Will be scalable and accessible to screen readers

    • Use W3 code validators to verify that it will run on multiple devices

      • Look for vscode extensions to check if your code is in compliance
      • Use developer tools to verify that your website renders correctly on different screen sizes
  7. The canvas element
    • It is the graphics element for HTML
    • Utilizes javascript to draw
  8. Navigation for mobile devices
    • Place navigation elements in easy to find places
    • Stick to text based hyperlinks and give more padding to the clickable area
    • Provide access to the full version of the site
    • Limit the content so the web version isn't overwhelming (think in terms of less real estate)
  9. Images for mobile
    • Image should resize proportionally as the view gets smaller
    • Image should rarely exceed the view of the width of the screen (fluid images)
    • Consider using WebGL to create 2d and 3d graphics

  10. HTML5 API's

    • DOM: Document Object Model manipulation

    • Canvas

      • The canvas tag
      • Uses DOM delectors to retrieve the canvas element and generate its context
      • Use the DOM element to start drawing!

    • Geolocation

      • Retrieves the devices GPS location through the Navigator object
      • You can set up a listeneer to watch the position using watchPosition()
      • Geolocation API WILL NOT GET YOUR LOCATION IF YOU DO NOT GIVE THE APP PERMISSION VIA YOUR BROWSER

    • Drag and Drop API

      • Incredibly powerful and cool tool to utilize
      • Can be very tedious as you handle every event that takes place

    • File API

      • Handling and manipulating file objects in JS
      • Creating a new instance of the File() class takes 2 or 3 parameters
        1. Data provided as an array of lines of text
        2. File name
        3. An object that can contain the file type, and lastModified date
        4. Example in JS file

    • History API

      • Reading and managing the browsers url and history

    • Should I do form validation?

      • Yes, you should
      • Input tags may take regex to validate without JS
        • Regex is a language that provides immediate form validation
      • Using JS, you can inject warnings or error messages into the form
      • Its good practice

      • Ex: bootstrap, material-ui, etc.

  15. GDPR (General Data Protection Regulation)
    • All websites that attract EU citizens must publish and follow a transparent privacy policy
    • If companies fail to do so, they will face penalties
    • Fines up to 10,000,000 EUR or 20,000,000 Euros
  16. COPPA (Childrens Online Privacy Protection Act
    • Protects US children under the age of 13
    • Require permission from parents for collection and use of personal information of minors
    • $40,000 per penalty (for each piece of data...for each person)
